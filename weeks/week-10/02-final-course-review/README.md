## Sample Review Questions

### HTML

1. What is the difference between `<article>` and `<section>`?
2. List five semantic HTML elements and describe when to use each.
3. What attributes would you add to a form input for basic client-side validation?
4. What does the `alt` attribute do, and why is it important?

### CSS

5. Explain the CSS box model. How does `box-sizing: border-box` change it?
6. How do you centre a div both horizontally and vertically with Flexbox?
7. What does `display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem;` do?
8. What is a media query? Write one that targets screens narrower than 600px.
9. What is the difference between `transition` and `animation`?

### JavaScript

10. What is the difference between `let`, `const`, and `var`?
11. Explain the difference between `==` and `===`.
12. Write an arrow function that takes two numbers and returns their sum.
13. How do you select an element by its ID? By its class? By a CSS selector?
14. What does `addEventListener` do? Give an example.
15. How does `async/await` work? What does it replace?

### Git & Deployment

16. What is the difference between `git pull` and `git fetch`?
17. How do you create a new branch and switch to it?
18. What is a pull request and when would you use one?
19. How do you deploy a static site with GitHub Pages?

### CSS Frameworks

20. What is the difference between Tailwind CSS and Bootstrap?
21. What does `class="m-4 p-2 text-center"` do in Tailwind?
22. How do you override a Bootstrap variable with Sass?

### Accessibility

23. What are the four principles of WCAG (POUR)?
24. What is the minimum colour contrast ratio for normal text (WCAG AA)?
25. Why should you never remove `outline` without providing an alternative focus style?
26. When should you use `aria-label` vs visible text?

### AI Integration

27. What is prompt engineering? Give three tips for writing effective prompts.
28. Why should you always verify AI-generated code?
29. How can AI help with accessibility audits?
30. What are some ethical concerns with AI-generated code?

## Q&A Format Guide

### Format

- **30 minutes of open Q&A** — students ask questions on any course topic
- Questions can be about concepts, code, final project, career advice
- Instructor answers with explanation, live demo, or code example
- Questions are moderated — one at a time, raise hand or use chat

### Popular Q&A Topics (Be Prepared)

| Category | Common Questions |
|----------|-----------------|
| Deployment | "My GitHub Pages site shows 404 — what's wrong?" "How do I point my own domain?" |
| Final project | "What's the minimum viable submission?" "Can I use a different CSS framework?" |
| CSS | "My Flexbox items aren't wrapping." "How do I overlay text on an image?" |
| Debugging | "My JavaScript isn't running." "CORS errors — how do I fix them?" |
| Career | "What should I learn after this course?" "How do I build a portfolio that gets me hired?" |

### Pro Tips for Students

- **Ask specific questions** — include code snippets, error messages, screenshots
- **One question per turn** — if you have multiple, queue them
- **Take notes** — answers may help your final project
- **No question is too basic** — if you're wondering, others probably are too

## Final Project Preparation Tips

### Project Requirements (Typical Rubric)

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Completeness | 25% | All required features implemented |
| Code quality | 20% | Semantic HTML, well-structured CSS, clean JS |
| AI integration | 20% | Meaningful use of AI tools (prompts, debugging, content) |
| Accessibility | 15% | WCAG AA, keyboard nav, contrast, ARIA |
| Deployment | 10% | Live URL, no broken links, functional |
| Documentation | 10% | README with setup, usage, AI tool reflections |

### Final Project Checklist

- [ ] Write a clear README with setup instructions
- [ ] Document which AI tools you used and how
- [ ] Test every feature end-to-end
- [ ] Check that all links work
- [ ] Run Lighthouse audit: Performance ≥ 80, Accessibility ≥ 90
- [ ] Validate HTML (W3C validator)
- [ ] Test on mobile (responsive layout)
- [ ] Test keyboard navigation on all interactive elements
- [ ] Resize to 200% zoom — no horizontal scroll
- [ ] Prepare a short (2 min) demo walkthrough

### Suggested Timeline

| When | What to Do |
|------|------------|
| Now | Review rubric, choose your project scope |
| Before session | Have a rough idea of your project structure |
| After session | Start building, focus on core feature first |
| 2 days before deadline | Polish, test, deploy |
| 1 day before deadline | Submit early — avoid last-minute issues |

### AI Tool Reflection Prompts

## Course Recap: Key Takeaways

1. **AI accelerates development** — use it for code generation, debugging, refactoring, and learning
2. **Always verify AI output** — AI can produce buggy, insecure, or inaccessible code
3. **Semantic HTML + CSS + JS** — the foundation of every web project
4. **Version control is non-negotiable** — Git tracks changes, enables collaboration, and powers deployment
5. **CSS frameworks save time** — utility-first (Tailwind) and component-based (Bootstrap) approaches both have strengths
6. **Accessibility is not optional** — build inclusive web experiences from the start
7. **Your portfolio is your best asset** — showcase your work, process, and growth
